URL (Uniform Resource Locator) — це стандартний формат, який використовується для вказівки адреси ресурсу в інтернеті. URL складається з кількох частин, кожна з яких відіграє певну роль у визначенні місцезнаходження ресурсу та способу доступу до нього.

Структура URL

  • Схема (Scheme):

    Вказує протокол, який використовується для доступу до ресурсу. Приклади: http, https, ftp, mailto.

    Приклад: https://

  • Ім'я хоста (Host):

    Вказує доменне ім'я або IP-адресу сервера, на якому розміщений ресурс.

    Приклад: www.example.com

  • Порт (Port):

    Вказує порт, через який здійснюється з'єднання з сервером. За замовчуванням для HTTP використовується порт 80, для HTTPS — 443.

    Приклад: :8080

  • Шлях (Path):

    Вказує шлях до конкретного ресурсу на сервері. Це може бути файл або каталог.

    Приклад: /path/to/resource

  • Параметри запиту (Query Parameters):

    Містять додаткові дані у вигляді пар "ключ-значення", які передаються на сервер. Починаються з символу ?, кожна пара розділяється символом &.

    Приклад: ?key1=value1&key2=value2

  • Фрагмент (Fragment):

    Вказує внутрішню частину ресурсу, наприклад, розділ на веб-сторінці. Починається з символу #.

    Приклад: #section2
